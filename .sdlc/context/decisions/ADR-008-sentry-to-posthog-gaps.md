# ADR-008: Sentry → PostHog Migration — Feature Gap Decisions

Date: 2026-05-21
Status: ACCEPTED
Replaces: ADR-004 (Sentry for all observability)
Generated by: `/sdlc:config mude do sentry para o posthog`

## Context

Sentry free tier does not support automatic GitHub issue creation, which was required
for the incident pipeline demo. PostHog free tier is chosen as replacement.
PostHog covers: analytics, session replay, error tracking (autocapture), feature flags.

## Feature Gap Decisions

### Source Maps
- **Sentry capability**: automatic source map upload via `@sentry/astro` Vite plugin — minified stacks resolved in dashboard
- **PostHog equivalent**: none — PostHog error tracking shows minified stacks without upload
- **Decision**: accept the trade-off; use Vercel deployment source maps tab for manual stack resolution
- **Trade-off**: debugging minified production errors requires an extra manual step

### Release Registration
- **Sentry capability**: `sentry-cli releases new/finalize` — tags errors to specific releases, enables before/after comparison
- **PostHog equivalent**: `posthog.capture('deployment', { version, environment })` event
- **Decision**: capture deployment event via CI `curl` call in `deploy.yaml`; version is available as an event property on all subsequent errors
- **Trade-off**: no native "first seen in this release" filter; must filter by event timestamp

### Sentry MCP (Incident Resolver)
- **Sentry capability**: Cursor Sentry MCP plugin — `sentry_find_issues`, structured issue ingestion
- **PostHog equivalent**: PostHog REST API (`/api/projects/{id}/errors/`) with personal API key
- **Decision**: incident-resolver agent queries PostHog API directly; requires `POSTHOG_PERSONAL_API_KEY` CI secret (not the public project key)
- **Trade-off**: no structured MCP — agent must parse raw JSON API response

### GitHub Issue Auto-Creation
- **Sentry capability**: Business/Enterprise plan — Sentry Alert → Create GitHub Issue action
- **PostHog equivalent**: PostHog Alert → Webhook → GitHub Actions `workflow_dispatch` (free tier)
- **Decision**: configure PostHog webhook to call GitHub Actions `incident-response.yaml` via `repository_dispatch`; the workflow already handles this event type
- **Trade-off**: webhook configuration must be done manually in PostHog UI; not captured as code

## Accepted Risks

- Minified stack traces in production error view (mitigated by Vercel source maps tab)
- `POSTHOG_PERSONAL_API_KEY` must be added as GitHub Secret for incident-resolver to query errors
- PostHog free tier has data retention limits (1 year); Sentry had 90-day default
